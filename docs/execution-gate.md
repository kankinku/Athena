# 실행 게이트

이 문서는 Athena가 계획 단계에서 실행 단계로 넘어가기 전에 확인해야 하는 조건을 정의한다.

## 목적

실행 게이트의 목적은 속도를 늦추는 것이 아니라 잘못된 실행을 막는 것이다.

## 기본 체크

- 목표와 현재 작업이 연결되는가
- 변경 범위가 명확한가
- 필요한 검증이 정의되어 있는가
- 롤백 또는 복구 경로가 있는가
- 정책상 허용되는가

## 사람 개입이 필요한 경우

- 공용 계약 변경
- DB 스키마 변경
- 보안 민감 경로 변경
- 고위험 원격 작업
- 롤백이 어려운 작업

## 중요한 원칙

실행 게이트는 제품 중심이 아니다. 자율 루프가 안전하게 다음 단계로 넘어가기 위한 경계선이다.
