# 비목표 (Non-Goals)

이 문서는 Athena 변환 프로젝트에서 **명시적으로 구축하지 않는 것**을 확정한다.
비목표의 명문화는 범위 확장(scope creep)을 방지하고 팀의 집중력을 유지하기 위해 필수적이다.

---

## 1. 범용 자율 연구 시스템 전면 구축 금지

**무엇을**: ML 학습 실험 자동화, 논문 탐색, 하이퍼파라미터 튜닝 등의 연구 기능을 신규 구축하는 것.

**이유**: 현재 Athena가 이미 이 기능을 제공하며, 새 시스템은 코드 변경 관리에 집중한다.

**경계**: 기존 연구 루프 기능은 유지하되, 새 협의 시스템과의 통합은 별도 마일스톤으로 다룬다.

---

## 2. 무제한 코드 수정 에이전트 구축 금지

**무엇을**: 임의의 코드를 제한 없이 수정하는 완전 자율 코딩 에이전트를 구축하는 것.

**이유**: 모듈 오너십과 실행 게이트가 이 시스템의 핵심 안전장치다. 이를 우회하는 에이전트는 시스템 목적에 반한다.

**경계**: 에이전트는 자신의 모듈 범위 내에서만, 합의를 거쳐서만 코드를 수정할 수 있다.

---

## 3. 완전 무인 운영 (Fully Unattended Operation)

**무엇을**: 운영자 없이 모든 결정과 실행이 자동으로 진행되는 완전 자율 시스템을 구축하는 것.

**이유**: 1차 목표는 **운영자 감독 하의 반자율 시스템**이다. 완전 무인 운영은 충분한 검증 이후의 별도 마일스톤이다.

**경계**: 
- 공용 인터페이스 변경: 운영자 승인 필수
- DB 스키마 변경: 운영자 승인 필수
- 보안 민감 경로 변경: 운영자 승인 필수
- 일반 내부 변경: 자동 실행 허용 (정책 설정 후)

---

## 4. 외부 논문/인용 엔진 우선 개발 금지

**무엇을**: 외부 학술 논문 수집, arXiv/Semantic Scholar 연동, 자동 인용 추적 시스템을 우선 구축하는 것.

**이유**: 현재 변환 프로젝트의 핵심은 코드 변경 협의 워크플로다. 외부 지식 수집은 후속 마일스톤이다.

**경계**: 기존 ingestion 시스템은 유지하지만, 신규 대규모 외부 연동은 이번 범위에 포함하지 않는다.

---

## 5. 멀티 저장소 동시 관리

**무엇을**: 여러 Git 저장소를 동시에 관리하고 교차 저장소 영향도를 분석하는 것.

**이유**: 1차 파일럿은 단일 저장소(Athena 자체)에서 시작한다. 멀티 저장소는 별도 마일스톤이다.

---

## 6. 실시간 스트리밍 협의 UI

**무엇을**: 에이전트 회의 과정을 실시간 스트리밍으로 보여주는 웹 대시보드.

**이유**: TUI 기반 CLI + Ink 컴포넌트가 현재 Athena의 UI 스택이다. 웹 UI는 별도 마일스톤이다.

**경계**: 현재 Ink TUI에 회의 상태를 추가하는 것은 목표에 포함된다 (Task 12).
