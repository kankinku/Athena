# 에이전트 소집 규칙

이 문서는 Athena가 언제 어떤 역할을 호출해야 하는지 정의한다.

## 기본 원칙

불필요한 에이전트를 부르지 않는다.

소집은 다음 질문에 답하기 위해 존재한다.

- 누가 현재 개선을 평가해야 하는가
- 누가 실행에 직접 참여해야 하는가
- 누가 위험을 검토해야 하는가

## 기본 분류

- 직접 영향 담당자: 실행 또는 검증에 직접 책임이 있는 역할
- 간접 영향 담당자: 계약, 의존성, 결과에 영향을 받는 역할
- 관찰자: 참고와 설명 가능성을 위해 필요한 역할

## 소집 시점

- planning에서 위험이 높아졌을 때
- executing 전에 범위 조정이 필요할 때
- evaluating에서 회귀나 충돌이 드러났을 때
- redesigning에서 방향 전환이 필요할 때

## 중요한 해석

소집은 Athena의 정체성이 아니다. 오케스트레이터가 다음 개선을 더 잘 정렬하기 위해 사용하는 도구다.
