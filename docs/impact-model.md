# 영향도 모델

이 문서는 영향도 분석이 Athena의 자율 개선 루프에서 어떤 역할을 하는지 정의한다.

## 영향도 분석의 목적

영향도 분석의 목적은 "회의 소집"이 아니다.

영향도 분석의 목적은 다음 세 가지다.

1. 어떤 모듈과 테스트가 영향을 받는지 파악
2. 다음 실행의 위험도와 검증 범위를 결정
3. 오케스트레이터가 어떤 작업 모드로 들어가야 하는지 판단

## 기본 분류

- `direct`: 변경이 직접 발생한 모듈
- `indirect`: 계약 또는 의존 관계로 영향받는 모듈
- `observer`: 참고만 필요한 주변 모듈

## 루프 안에서의 역할

- planning 단계에서는 범위와 위험을 정한다.
- executing 단계에서는 어떤 파일과 모듈을 건드릴지 제한한다.
- evaluating 단계에서는 어떤 테스트와 회귀 경로를 봐야 하는지 정한다.

## 중요한 원칙

영향도 분석은 제품의 주인공이 아니다. 자율 개선이 목표에서 벗어나지 않도록 잡아주는 보조 모델이다.
