# 운영자 런북

이 문서는 운영자가 Athena의 자율 루프를 관찰하고 개입하는 방법을 설명한다.

## 운영자의 역할

운영자는 Athena를 직접 대신해서 일하는 사람이 아니다.

운영자는 다음을 관리한다.

- 목표와 우선순위
- 정책과 위험 한계
- 중단 및 승인 시점
- 실패 시 복구 경로

## 일상 점검

```bash
athena research runs
athena research next-actions
athena research workflow <run-id>
athena report <run-id>
```

## 언제 개입해야 하는가

- 고위험 변경이 포함될 때
- 정책 위반이 감지될 때
- 검증 실패가 반복될 때
- 롤백이 어려운 작업이 실행되려 할 때
- 루프가 목표에서 벗어날 때

## 중요한 원칙

운영자 감독은 Athena의 본질이 아니라 현재 가장 잘 검증된 운영 모드다. 운영자의 역할은 루프를 대신 수행하는 것이 아니라 루프의 안전과 방향을 보증하는 것이다.
