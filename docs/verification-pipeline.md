# 검증 파이프라인

이 문서는 Athena의 `evaluate` 단계를 구조화한다.

## 목적

검증은 실행이 끝났는지 확인하는 절차가 아니다. 개선이 실제로 유효했는지 판단하는 절차다.

## 기본 순서

1. 변경된 모듈의 직접 검증
2. 계약 및 의존성 검증
3. 통합 흐름 검증
4. 필요 시 E2E 또는 원격 경로 검증

## 평가가 답해야 하는 질문

- 목표에 더 가까워졌는가
- 회귀가 생기지 않았는가
- 결과를 설명할 근거가 있는가
- 다음 반복으로 무엇을 가져가야 하는가

## 실패 시

- 단순 실패면 재설계로 돌아간다.
- 범위 충돌이나 고위험 실패면 협의 또는 승인 경로로 올라간다.
